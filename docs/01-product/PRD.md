# 01 PRD

## 产品目标

在 MVP 阶段实现一个可运行、可体验、可验证付费潜力的 Windows AI 桌宠产品。

用户应能完成以下闭环：

1. 上传宠物主图，可选补充参考图。
2. 创建宠物身份。
3. 生成 Pet DNA。
4. 在桌面看到宠物。
5. 与宠物进行基础互动。
6. 通过喂食、铲屎、聊天影响宠物状态。
7. 看到宠物状态和关系随时间变化。

## MVP 范围

MVP 只做一个清晰闭环，不追求完整平台化。

包含：

- 宠物创建
- Pet DNA
- 桌宠运行
- 喂食
- 铲屎
- 抚摸
- 基础聊天
- 情绪、饱食、亲密度、经验值
- 本地与后端同步

不包含：

- 创作者市场
- 插件 SDK
- 多人社交
- 复杂 3D
- 完整支付系统
- 年度绘本
- 高级视频生成

## 核心模块

### Pet Studio

负责创建宠物数字身份。

用户上传 1 张主图和最多 4 张参考图后，AI 识别宠物类型、颜色、品种、体型、眼睛、耳朵、尾巴、毛色、花纹等特征，并生成初始 Pet DNA。

### Desktop Pet

负责桌面呈现和互动。

宠物在桌面上可以待机、行走、睡觉、进食、跳跃、看鼠标、被拖动、被抚摸。桌宠运行时不直接承载复杂业务逻辑，只消费后端或本地状态。

### Pet Life

负责状态成长。

维护饱食度、心情、能量、清洁度、亲密度、等级、经验等状态。用户行为会改变状态，系统也会按时间自然衰减或恢复。

### AI Companion

负责对话、主动提醒和人格表达。

MVP 阶段只支持基础聊天和少量主动短句。长期版本会加入记忆召回、工作状态理解和 Pet Story。

## 用户故事

作为宠物主人，我希望上传一张清晰主图，并可补充侧面、背面或局部参考图，系统能更准确地生成一只像我家宠物的桌宠，让我感觉它在陪我。

作为长时间工作的人，我希望宠物能提醒我喝水、休息，而不是只会播放动画。

作为养成类用户，我希望喂食、铲屎、抚摸能影响宠物状态，让我产生照顾它的感觉。

作为付费用户，我希望宠物能够记住我的习惯、重要日期和互动历史。

## 验收标准

- 用户可在 5 分钟内完成首次宠物创建。
- 桌宠可在 Windows 桌面透明置顶显示。
- 用户至少能完成喂食、铲屎、抚摸、聊天四类互动。
- 每次互动都能立即看到 UI 反馈。
- 宠物状态变化可保存并恢复。
- API 错误时前端展示明确错误态，不出现白屏。
