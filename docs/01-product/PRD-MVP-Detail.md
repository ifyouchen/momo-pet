# PRD MVP Detail

## 目标

本文档把 MVP 中必须实现的功能拆到页面、字段、状态和异常级别，作为后续编码的直接依据。

## MVP 功能边界

必须实现：

- 首次启动引导
- 创建宠物
- 上传宠物照片
- AI 生成 Pet DNA
- 用户确认或编辑 Pet DNA
- 部署桌宠
- 查看宠物状态
- 喂食
- 铲屎
- 抚摸
- 基础聊天
- 本地与后端状态保存

不实现：

- 宠物交友
- 商城
- 支付
- 插件系统
- 多端同步
- 年度绘本
- 高级视频动作生成

## 创建宠物

### 字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| name | string | 是 | 宠物名称，1 到 20 个字符 |
| species | enum | 是 | CAT、DOG、BIRD、RABBIT |
| birthday | date | 否 | 用户可不填，默认使用创建日期 |
| description | string | 否 | 用户对宠物性格或外观的描述 |

### 规则

- name 不能为空。
- species 可以由 AI 识别，也可以由用户手动选择。
- 用户跳过照片时只能进入手动创建流程，不能调用照片识别任务。

### 异常

- 名称过长：提示“名字太长啦，换个短一点的名字吧。”
- 未选择类型：提示“先告诉我它是哪种宠物。”
- 网络失败：保留表单内容，允许重试。

## 照片上传

### 支持格式

- JPG
- PNG
- WebP

### 限制

- 单张照片不超过 10 MB。
- MVP 支持 1 张主图 + 最多 4 张参考图。
- 单次创建流程最多 5 张图片，批次总大小不超过 30 MB。
- 主图必填，参考图选填。
- 参考图必须来自同一只宠物，用于提升外观、花纹、体型和局部特征识别准确度。

### 状态

- EMPTY：未上传。
- UPLOADING：上传中。
- UPLOADED：上传成功。
- FAILED：上传失败。
- MISMATCH_WARNING：AI 判断参考图可能不是同一只宠物，需要用户确认。

### 异常

- 格式不支持：提示支持格式。
- 文件过大：提示大小限制。
- 图片数量超限：提示“最多上传 1 张主图和 4 张参考图。”
- 上传失败：保留选择入口，允许重试。

## AI 生成 Pet DNA

### 状态

- PENDING：任务已创建。
- RUNNING：生成中。
- SUCCEEDED：生成成功。
- FAILED：生成失败。
- CANCELED：用户取消。

### 成功后展示

- 名称
- 类型
- 品种
- 主色
- 花纹
- 眼睛颜色
- 体型
- 性格
- 精力
- 喜欢的食物
- 讨厌的事物
- 口头禅
- 识别置信度
- 多图不一致提醒

所有字段允许用户编辑。

### 失败处理

展示：

- 重试按钮
- 手动填写按钮
- 返回上传按钮

AI 失败不能阻断用户创建宠物。

## 部署桌宠

### 成功条件

- 宠物档案已创建。
- Pet DNA 已确认。
- 默认状态已初始化。
- 桌宠资源可加载，或可使用默认资源。

### 结果

桌宠透明窗口出现，宠物进入 Idle 状态，并弹出首次引导气泡。

## 养成互动

### 喂食

用户选择食物后：

- 播放 Eat 动作。
- hunger 增加。
- mood 小幅增加。
- experience 增加。
- 如果已过饱，拒绝喂食。

### 铲屎

存在清理事件时：

- 用户点击清理。
- cleanliness 增加。
- mood 小幅增加。
- experience 增加。

没有清理事件时：

- 不调用后端。
- 前端提示“现在很干净。”

### 抚摸

用户点击抚摸：

- intimacy 增加。
- mood 增加。
- 播放 Happy 动作。

连续高频点击：

- 根据 personality 决定开心或轻微不满。

## 基础聊天

### 输入

- 文本长度不超过 500 字。
- 空文本不能发送。

### 输出

- 宠物口吻回复。
- 回复不超过 80 个中文字符。
- 失败时保留用户输入，可重试。

## 状态面板

必须展示：

- 名称
- 等级
- 饱食度
- 心情
- 清洁度
- 能量
- 亲密度

每个状态都需要数字和可读标签。

## MVP 验收

- 用户能在 5 分钟内完成首次创建。
- AI 失败时仍可手动创建。
- 四个互动行为都有即时反馈。
- 重启应用后宠物状态不丢失。
- 后端错误不会导致前端白屏。
