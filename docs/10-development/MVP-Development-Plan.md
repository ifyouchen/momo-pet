# MVP Development Plan

## 目标

本计划适用于“一名 AI 开发执行 + 一名人工验收”的协作模式。

目标不是一次性做完整产品，而是按可验收的小闭环逐步推进 Project Momo MVP。

## 产品依据

本计划必须服从以下产品文档：

- `docs/00-project/Vision.md`
- `docs/00-project/Roadmap.md`
- `docs/01-product/PRD.md`
- `docs/01-product/PRD-MVP-Detail.md`
- `docs/01-product/MVP.md`
- `docs/01-product/FeatureList.md`

如果本计划与上述产品文档冲突，以产品文档为准，先更新产品文档再调整开发计划。

## 协作角色

### AI 开发执行

职责：

- 阅读相关文档。
- 拆解当前任务。
- 编写代码。
- 自测。
- 汇报变更和风险。
- 不擅自扩大 MVP 范围。

### 人工验收

职责：

- 确认功能是否符合 MVP。
- 按验收用例测试。
- 反馈阻塞问题。
- 决定是否进入下一步。

## 开发原则

- 每次只推进一个可验证目标。
- 优先打通闭环，再优化体验。
- 先默认 Momo Cat，再接 AI 生成。
- 先本地可运行，再做复杂云能力。
- 不做宠物交友、商城、支付、插件系统。

## 阶段计划

### Sprint 0：工程骨架

目标：

- 项目能启动。
- 后端、桌面端、后台目录清晰。
- 基础规范落地。

交付：

- `apps/backend`
- `apps/desktop`
- `apps/admin`
- `assets/momo-cat`
- README 启动说明
- 基础 lint/format 配置

验收：

- 后端能启动并返回健康检查。
- 桌面端能启动空窗口。
- Admin 能启动空页面。

### Sprint 1：默认宠物闭环

目标：

- 使用默认 Momo Cat 跑通宠物状态和照顾行为。

交付：

- Pet 创建接口。
- PetState 初始化。
- Feed / Touch / Clean API。
- 前端宠物主页静态实现。
- 状态变化展示。

验收：

- 能创建默认宠物。
- 能看到状态条。
- 喂食改变饱食度。
- 抚摸改变亲密度。
- 铲屎改变清洁度。

### Sprint 2：桌面端运行时

目标：

- 桌宠在 Tauri 中可见、可拖动、可恢复位置。

交付：

- Tauri 透明窗口。
- Momo Cat 显示。
- 窗口拖动。
- 本地缓存窗口位置。
- 显示/隐藏。

验收：

- 桌宠窗口可打开。
- 拖动后重启位置恢复。
- 隐藏后可恢复。

### Sprint 3：宠物交互模式

目标：

- 把按钮操作升级成鼠标交互模式。
- 该 Sprint 是喂食、抚摸、铲屎三个 MVP 照顾行为的体验实现方式，不新增产品功能范围。

交付：

- FeedingMode：拖食物给宠物。
- PettingMode：鼠标滑动抚摸。
- CleaningMode：点击清理事件。
- 交互成功反馈。
- 交互失败提示。

验收：

- 食物拖到宠物附近后触发喂食。
- 抚摸滑动 1 秒后增加亲密度。
- 清理事件点击后消失。
- Esc 或超时可以退出交互模式。

### Sprint 4：Pet Studio

目标：

- 用户可以创建宠物并确认 Pet DNA。
- 上传流程支持 1 张主图和最多 4 张参考图。

交付：

- 创建宠物页面。
- 主图和参考图上传页面。
- AI 分析模拟流程。
- Pet DNA 确认页面。
- 手动 Pet DNA 创建。

验收：

- AI 不接入时也能手动创建宠物。
- 用户必须选择 1 张主图，参考图最多 4 张。
- 超过数量、格式不支持、单文件过大时有明确错误提示。
- Pet DNA 可以展示和编辑。
- 确认后进入宠物主页。

### Sprint 5：AI Pet DNA 任务

目标：

- 接入异步 AI 任务生成 Pet DNA 草稿。

交付：

- 单图上传接口，支持 photoRole。
- Pet DNA 任务支持 primaryPhotoAssetId 和 referencePhotoAssetIds。
- AI 任务创建。
- AI 任务状态查询。
- Pet DNA 草稿生成。
- AI 失败降级为手动创建。

验收：

- 上传 1 张合法主图后可以生成任务。
- 上传主图和参考图后，任务记录包含完整图片资源引用。
- 参考图超过 4 张时任务创建失败并返回明确错误码。
- 任务成功后展示 Pet DNA 草稿。
- 任务失败后可以手动创建。

### Sprint 6：基础聊天

目标：

- 用户能和宠物进行基础聊天。

交付：

- Chat API。
- Chat Panel。
- 宠物口吻回复。
- AI 失败降级回复。
- 最近聊天记录。

验收：

- 发送消息后有回复。
- 回复符合宠物口吻。
- AI 失败不白屏，输入不丢失。

### Sprint 7：后台 MVP

目标：

- 能查看关键数据和 AI 任务。
- 后台是内部排障和验收支撑工具，不是用户侧 MVP 核心闭环。若核心闭环延期，本 Sprint 可顺延。

交付：

- Dashboard。
- Pets 列表。
- Pet Detail。
- AI Tasks 列表。
- 失败任务查看。

验收：

- 可以查宠物。
- 可以查 AI 任务。
- 可以看到失败原因。

### Sprint 8：Beta 验收

目标：

- 修复 MVP 主流程问题，准备小范围体验。

交付：

- MVP 验收用例通过。
- 错误文案整理。
- 本地缓存稳定。
- 基础打包。

验收：

- P0 验收用例全部通过。
- Windows 桌宠体验连续使用 30 分钟无明显异常。

## 不做清单

MVP 不做：

- 宠物交友。
- 商城。
- 支付。
- 插件 SDK。
- 多端同步。
- 复杂 Sprite AI 生成。
- 语音。
- Pet World。

## 进入下一 Sprint 条件

- 当前 Sprint P0 验收项通过。
- 没有阻塞主流程的 bug。
- README 或相关文档已更新。
- 人工验收确认。
