# Design System

## 设计目标

Project Momo 的 UI 应该安静、温暖、轻量，避免复杂运营后台感，也避免幼稚化。

## 视觉关键词

- 温暖
- 清晰
- 陪伴
- 克制
- 低打扰

## 组件原则

- 桌面端面板保持小尺寸。
- 操作按钮使用图标加短文本。
- 状态数字必须易读。
- 错误态和空态必须明确。

## MVP 组件

- PetStatusPanel
- CareActionBar
- ChatPanel
- UploadPhotoPanel
- PetDnaConfirmPanel
- AiTaskProgress
- SettingsPanel

## 色彩

使用语义色而不是装饰色：

- Primary：主要操作。
- Success：生成成功、喂食成功、同步成功。
- Warning：饥饿、清洁度下降、生成耗时。
- Danger：上传失败、任务失败。
- Neutral：面板、文本、边框。

## 字体与文案

- 字体以可读性优先。
- 宠物说话要短。
- 系统提示要明确。
- 不把技术术语直接暴露给普通用户。

## 间距与布局

- 桌面面板紧凑但不拥挤。
- 操作按钮保持稳定尺寸。
- 不使用多层嵌套卡片。
- 不做营销式大留白。

## 图标

MVP 图标包括上传、喂食、清理、抚摸、聊天、设置、隐藏、退出。

图标必须配合文本或 tooltip，避免含义不明。

## 桌面 UI

桌宠本体不叠加复杂 UI，只允许短句气泡、状态变化反馈和轻量提醒。点击宠物打开互动面板，托盘菜单提供显示、隐藏、设置和退出。
