# Pet Operation Page Spec

## 目标

宠物操作页面是用户管理和照顾桌宠的主界面。它不是后台页面，而是一个轻量的宠物控制台，用来承载状态查看、喂食、铲屎、抚摸、聊天和桌宠设置。

## 页面定位

用户打开应用后，如果已经有宠物，默认进入宠物操作页面，并同步显示桌宠窗口。

页面需要回答三个问题：

- 我的宠物现在怎么样？
- 我现在可以对它做什么？
- 它今天发生了什么？

## 页面布局

```text
┌────────────────────────────────────────────────────────┐
│ Header：宠物头像 / 名字 / 等级 / 当前情绪 / 设置入口     │
├──────────────────────────┬─────────────────────────────┤
│ Pet Preview              │ Status & Actions             │
│                          │                             │
│ - 桌宠动画预览            │ - 饱食度                      │
│ - 当前气泡短句            │ - 心情                        │
│ - 当前行为状态            │ - 清洁度                      │
│                          │ - 能量                        │
│                          │ - 亲密度                      │
│                          │                             │
│                          │ [喂食] [铲屎] [抚摸] [聊天]   │
├──────────────────────────┴─────────────────────────────┤
│ Today Timeline：今日事件 / 成长记录 / 最近聊天摘要        │
└────────────────────────────────────────────────────────┘
```

## Header

### 字段

| 字段 | 说明 |
| --- | --- |
| avatar | 宠物头像或默认头像 |
| name | 宠物名称 |
| level | 当前等级 |
| emotion | 当前情绪 |
| statusText | 一句当前状态文案 |

### 操作

- 切换宠物：MVP 可隐藏，Beta 支持。
- 打开设置。
- 显示或隐藏桌宠。

## Pet Preview

### 展示内容

- 当前 Sprite 动画。
- 当前行为：Idle、Walk、Sleep、Eat、Happy。
- 宠物气泡短句。
- 清理事件提示。

### 状态

| 状态 | 表现 |
| --- | --- |
| Loading | 展示骨架屏和“正在叫醒宠物” |
| Ready | 播放当前动作 |
| AssetFailed | 使用默认 Sprite |
| Offline | 显示最近一次缓存状态 |

### 约束

- 动画区域尺寸稳定，不能随帧变化跳动。
- 不在预览区堆叠复杂 UI。
- 气泡短句不超过 20 个中文字符。

## Status & Actions

### 状态条

| 状态 | 范围 | 展示 |
| --- | --- | --- |
| hunger | 0-100 | 数字 + 进度条 + 饱食标签 |
| mood | 0-100 | 数字 + 进度条 + 情绪标签 |
| cleanliness | 0-100 | 数字 + 进度条 |
| energy | 0-100 | 数字 + 进度条 |
| intimacy | 0-100 | 数字 + 进度条 |

### 操作按钮

| 操作 | 图标 | 后端接口 | 成功反馈 |
| --- | --- | --- | --- |
| 喂食 | food | POST /care/feed | Eat 动作，状态增量 |
| 铲屎 | clean | POST /care/clean | 清理事件消失 |
| 抚摸 | hand | POST /care/touch | Happy 动作 |
| 聊天 | chat | POST /chat/messages | 打开聊天面板 |

### Loading

按钮请求中：

- 当前按钮 disabled。
- 显示 loading。
- 其他按钮可按业务需要保持可用。

### Error

错误文案必须宠物化但不掩盖原因：

- 已吃饱：`我真的吃不下啦。`
- 无清理事件：`现在很干净。`
- 网络失败：`网络好像断开了，稍后再试。`

## Feed Panel

### 食物

MVP 食物：

- DRIED_FISH：小鱼干
- CHICKEN：鸡胸肉
- MILK：牛奶
- SNACK：零食

### 展示字段

- 食物名称
- 效果预览
- 是否可用

### 规则

- hunger >= 95 时食物按钮禁用。
- 食物点击后关闭选择面板，并播放 Eat。

## Clean Interaction

### 有清理事件

展示清理提示，用户点击后调用接口。

成功后：

- cleanliness 增加。
- 清理提示消失。
- 宠物播放 Happy 或 Idle。

### 无清理事件

不进入错误弹窗，只展示轻提示。

## Touch Interaction

点击抚摸后立即给轻反馈，再等待后端确认。

成功：

- intimacy 增加。
- mood 增加。
- 播放 Happy。

失败：

- 回滚乐观反馈。
- 展示轻提示。

## Chat Panel

```text
┌──────────────────────────────┐
│ 和奶糖聊天                    │
│                              │
│ 奶糖：今天也要陪我玩哦。       │
│ 你：今天工作好累              │
│ 奶糖：辛苦啦，我陪你休息一下。 │
│                              │
│ [ 输入消息...              ]  │
│                      [ 发送 ] │
└──────────────────────────────┘
```

### 状态

- Empty：展示宠物欢迎语。
- Typing：用户输入。
- Sending：发送中。
- Failed：保留输入并允许重试。
- Offline：禁止发送，展示离线提示。

### 规则

- Enter 发送。
- Shift + Enter 换行。
- 单条消息不超过 500 字。
- 回复不超过 80 个中文字符。

## Today Timeline

展示今日最多 10 条事件。

事件类型：

- 创建宠物
- 喂食
- 铲屎
- 抚摸
- 聊天
- 升级
- AI 生成成功

事件文案示例：

```text
10:30 奶糖吃了小鱼干，心情变好了。
11:05 你摸了摸奶糖，亲密度提升。
```

## Settings Entry

MVP 设置项：

- 显示或隐藏桌宠
- 开机启动
- 主动提醒
- 减少动画
- 清理本地缓存
- 退出应用

## 响应式规则

桌面主窗口：

- 推荐宽度：960 到 1200 px。
- Pet Preview 与 Status 并排。

窄窗口：

- Pet Preview 在上。
- Status & Actions 在下。
- Timeline 折叠。

## 验收标准

- 用户能从该页面完成喂食、铲屎、抚摸、聊天。
- 所有操作都有 loading、成功和失败反馈。
- 网络失败不白屏。
- 离线模式能展示最近状态。
- 状态数值变化与后端返回一致。
