# Roadmap

## 目标

Roadmap 定义 Project Momo 从 MVP 到商业版本的阶段边界，避免在早期把 Pet World、商城、插件系统等远期能力提前压进开发。

## Phase 0：文档与工程骨架

交付：

- 核心产品文档
- AI 系统设计
- 后端 DDD 架构
- Tauri + React 桌面端骨架
- Sprint 拆分

## Phase 1：MVP

周期：4 到 8 周。

交付：

- 上传宠物照片
- 创建宠物档案
- Pet DNA 生成与确认
- 透明桌宠窗口
- Idle、Walk、Sleep、Eat、Happy 动作
- 喂食、铲屎、抚摸
- 基础聊天
- 状态保存

## Phase 2：Beta

周期：2 到 3 个月。

交付：

- AI 记忆摘要
- 主动提醒
- 开机启动
- 多只宠物
- 云同步
- 安装包与更新机制
- 基础埋点

## Phase 3：商业化

交付：

- 订阅体系
- 高级动作包
- 高质量资源重生成
- Pet Story 日记
- 相册和时间线
- 付费权益控制

## Phase 4：平台化

交付：

- 插件 SDK
- 创作者市场
- Pet World
- 多设备同步
- 社交访问
